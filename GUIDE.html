🏠 TerangaLoc

Guide d'utilisation

Plateforme de location de logements au Sénégal — annonces vérifiées, recherche par quartier, visite immersive 360°, messagerie et offres de visibilité Premium.

Guide complet — de l'installation à l'utilisation de chaque fonctionnalité.

1. Architecture du projet

terangaloc/
├── backend/                 API Laravel (PHP) — authentification, quartiers, profils
├── frontend/                Site principal (HTML/CSS/JS statique) ← l'application
│   ├── accueil.html         Page d'accueil + recherche
│   ├── recherche.html       Résultats + carte + filtres
│   ├── logement.html        Fiche d'un logement + visite 360°
│   ├── publier.html         Publier une annonce (propriétaire)
│   ├── connexion.html       Connexion / Inscription
│   ├── tableau-de-bord.html Espace personnel
│   ├── messagerie.html      Messagerie
│   ├── abonnement.html      Offres Premium
│   └── assets/              api.js, app.js, theme.css, pano/ (panoramas 360°)
├── frontend-react-legacy/   Ancienne version React (archivée, non utilisée)
└── docker-compose.yml       Base de données PostgreSQL + Mailpit

2. Prérequis

À installer une seule fois sur la machine :

  • PHP 8.2+ et Composer (backend Laravel)
  • Docker Desktop (base de données PostgreSQL)
  • VS Code + extension Live Server, ou tout serveur statique (php -S, npx http-server…)
  • Un navigateur récent (Chrome, Edge, Firefox)

3. Installation (première fois)

a) Récupérer le projet

git clone https://github.com/momo2889/LOCATION-LOGEMENT.git
cd LOCATION-LOGEMENT

b) Base de données (Docker)

docker compose up -d

Démarre PostgreSQL (port 5433) et Mailpit (emails de test : http://localhost:8025).

c) Backend (Laravel)

cd backend
composer install
cp .env.example .env          # créer le fichier d'environnement
php artisan key:generate      # générer la clé de l'application
php artisan migrate --seed    # créer les tables + données de démo

d) Frontend

Aucune installation : c'est un site statique, rien à compiler.

4. Démarrer l'application (chaque session)

Ouvrir 3 choses :

  1. La base de données (si pas déjà lancée) : docker compose up -d
  2. Le serveur backend (dans backend/) : php artisan serve → API sur http://127.0.0.1:8000
  3. Le site (dans frontend/) : clic droit sur accueil.html → Open with Live Server, ou php -S 127.0.0.1:5500 puis ouvrir http://127.0.0.1:5500/accueil.html
⚠️ Toujours passer par un serveur (http://…), jamais en file://, sinon les appels à l'API sont bloqués.

6. Utilisation, fonctionnalité par fonctionnalité

🏠 Accueil

  • Barre de recherche : choisir un quartier, un type, un budget max, puis Rechercher → résultats déjà filtrés.
  • Section « Logements vedettes » : les meilleures annonces mises en avant.

🔍 Recherche

  • Filtres : texte (quartier/type), type, budget max, durée (longue/courte), tri (pertinence, prix, surface).
  • La carte affiche les prix ; survoler une annonce met en évidence son point.

🖼️ Fiche logement

  • Visite immersive 360° : faire glisser pour explorer l'intérieur ; plein écran pour l'effet casque VR.
  • Galerie, caractéristiques (chambres, surface, meublé…), description.
  • « Contacter le propriétaire » → ouvre la messagerie avec un message pré-rempli sur ce logement.

👤 Connexion / Inscription

  • Connexion : email + mot de passe.
  • Inscription : choisir l'espace (Locataire 🔑 ou Propriétaire 🏠), puis nom, email, téléphone et mot de passe (8 caractères min., avec majuscule, minuscule et chiffre).
  • Une fois connecté : nom + bouton Déconnexion dans le menu.

📊 Tableau de bord

  • Accès réservé aux personnes connectées (sinon redirection).
  • Salutation personnalisée, statistiques et graphiques.

💬 Messagerie

  • Conversations à gauche, discussion à droite.
  • Arrivée depuis une fiche : conversation avec le bon propriétaire ouverte automatiquement.

➕ Publier une annonce

  • Réservé aux personnes connectées.
  • 3 étapes : Informations (titre, type, quartier, loyer…), Photos & 360°, Vérification et envoi.
  • La liste des quartiers vient du backend.

⭐ Abonnement Premium (côté propriétaire)

  • 3 offres : Gratuit, Premium, Pro/Agence.
  • Une annonce Premium est mise en avant (remonte en tête des résultats) — de façon transparente pour le locataire, sans badge visible.

🌙 Confort

  • Bouton 🌙 / ☀️ : thème clair / sombre (mémorisé).
  • Ctrl + K : recherche rapide (pages, quartiers, logements).
  • Cœur 🤍 : ajouter aux favoris.

7. Ce qui est réel vs. démonstration

Branché au backend (données réelles, persistantes) :

  • Inscription, connexion, déconnexion, mot de passe oublié
  • Tableau de bord personnalisé (utilisateur connecté)
  • Liste des quartiers

Encore en données de démonstration (backend à construire) :

  • Les logements (recherche, fiche, publication) et la messagerie utilisent des données d'exemple locales.
  • L'abonnement Premium et le paiement sont une démonstration (Wave / Orange Money / carte à intégrer plus tard).
